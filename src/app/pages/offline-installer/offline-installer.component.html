For maintainers building the native Windows installer (WiX) and testing API + UI on one port. Run these steps on a Windows build machine with Node.js, WiX Toolset v3.11+ (with Bal), and optional EDB PostgreSQL / Node MSIs for the Burn bundle.

1. Web (Angular)

Production build with apiUrl pointing at http://localhost:8080.

cd 3sm-web
npm ci
npm run build:production

Output: dist/3sm-web/ (or your project output path).

2. API (Nest)

Compile the API; for local smoke test, serve static UI from the same process.

cd 3sm-api
npm ci
npm run build

One-port dev (after web build): copy .env.oneport.example to .env.oneport, set S3M_STATIC_WEB_ROOT to the web dist folder, then:

npm run start:oneport

Open http://localhost:8080/setup to verify import flows.

3. WiX installer

From the API repo, on Windows:

  • Download NSSM and optional prerequisites/postgresql-installer.exe, prerequisites/node-lts.msi (see deploy/offline/wix/prerequisites/README.txt).
  • Run deploy\offline\wix\build.ps1 with -WebDist, -ApiDist, -NssmExe (see deploy/offline/wix/README.md).
cd deploy\offline\wix
.\build.ps1 -WebDist "C:\path\to\3sm-web\dist\3sm-web" `
  -ApiDist "C:\path\to\3sm-api\dist" `
  -NssmExe "C:\path\to\nssm.exe"

Produces Product.msi and S3M-Offline-Burn-Setup.exe (unless -SkipBundle). Upgrade rules: UPGRADES.md. Full checklist: NATIVE-ONLY-ROADMAP.md.
